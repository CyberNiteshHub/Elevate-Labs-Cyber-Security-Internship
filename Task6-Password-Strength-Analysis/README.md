# 🔐 Password Strength Analysis Report

## 📌 Overview

This project focuses on analyzing password strength using an online password evaluation tool.
The goal is to understand how different types of passwords perform against security standards and how complexity impacts overall strength.

---

## 🎯 Objective

* Analyze different passwords based on:

  * Length
  * Character variety (uppercase, lowercase, numbers, symbols)
  * Complexity
* Identify weak vs strong passwords
* Understand common password attacks
* Learn best practices for secure password creation

---

## 🛠️ Tools Used

* Web Browser (Kali Linux)
* Online Tool: **passwordmeter.com**
* Screenshot Analysis
* Report Generation (PDF)

---

## 🧪 Testing Methodology

Different types of passwords were tested:

| Password              | Score | Strength    |
| --------------------- | ----- | ----------- |
| 123                   | 4%    | Very Weak   |
| person123             | 41%   | Good        |
| Person@123            | 87%   | Very Strong |
| Cyber@Person2026      | 100%  | Very Strong |
| Cyb3r!P3rson#Secure99 | 100%  | Very Strong |
| MyDogNameIsRocky@2026 | 100%  | Very Strong |

---

## 📊 Key Observations

* ❌ Short passwords are extremely weak
* ⚠️ Only numbers or simple words are insecure
* ✅ Adding uppercase + symbols increases strength
* 🚀 Long and complex passwords achieve maximum security

---

## ⚠️ Common Password Attacks

* **Brute Force Attack**
  Tries all possible combinations until correct password is found

* **Dictionary Attack**
  Uses common password lists to crack passwords quickly

---

## 🔒 Best Practices for Strong Passwords

* Use at least **12–16 characters**
* Include:

  * Uppercase letters
  * Lowercase letters
  * Numbers
  * Symbols
* Avoid:

  * Personal names
  * Birthdates
  * Common words
* Use **passphrases** instead of simple passwords
